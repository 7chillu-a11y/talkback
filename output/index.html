리서치가 발견한 진짜 욕구

데이터 너머, 사람들이 진짜 원하는 것을 발견했습니다.

🎨
INSIGHT 01
사람들은 '정보'가 아니라 '그 시절의 분위기'를 소장하고 싶어 한다

MBTI 분석 결과보다, 우리가 그때 어떤 말투로 웃었는지가 더 소중합니다. 숫자는 잊혀도, 느낌은 남습니다.

⚡
INSIGHT 02
'언젠가 보고 싶다'는 감정이 행동의 핵심 트리거다

기념일, 졸업, 이별, 이직 — 인생의 전환점마다 과거 대화를 다시 보고 싶은 충동이 급증합니다. 이 순간이 바로 톡백의 사용 맥락입니다.

💎
INSIGHT 03
백업은 '의무'가 아니라 '감성 경험'이 되어야 한다

"저장해야지"가 아닌 "이걸 꼭 남기고 싶다"는 감정적 동기가 필요합니다. 톡백은 기술적 백업이 아닌 감성적 아카이빙으로 설계합니다.

당신의 추억은 암호 뒤에 안전하게 존재합니다

우리는 당신의 대화를 '분석'하지 않습니다. '당신이 원하는 것'을 분석합니다 — 3가지 설계 원칙이 이것을 증명합니다.

🔐
원칙 01 · On-Device 비식별화

당신의 대화는
기기를 떠나기 전에
이미 암호화됩니다

서버로 전송되기 전, 기기 내에서 이름·전화번호가 분석 ID로 치환됩니다. 원본 식별 정보는 기기 밖으로 나가지 않습니다.

✓ 기기 내 비식별화 후 전송
✓ 이름 · 번호 → 익명 ID 치환
✓ 원본은 서버가 볼 수 없습니다
🤝
원칙 02 · 상호 동의 시스템

혼자서는
발행할 수 없습니다.
함께여야 가능합니다

백서 발행 시 대화 참여자 전원의 동의가 필수입니다. 동의 철회 즉시 발행이 중단됩니다. 내 기록을 상대방이 일방적으로 공개할 수 없습니다.

✓ 발행 전 참여자 전원 동의 필수
✓ 동의 철회 시 즉시 비공개 처리
✓ 일방적 공개 구조적으로 불가
💨
원칙 03 · 휘발성 저장 원칙

분석이 끝나면
원본 텍스트는
사라집니다

AI 분석 완료 후 원본 대화 텍스트는 즉시 파기됩니다. 서버에는 분석 결과값만 암호화 저장. 서버는 처음부터 원본을 보지 않습니다.

✓ 분석 완료 즉시 원본 파기
✓ 서버 저장 = 결과값만 암호화
✓ 원본 재요청 불가 (설계상 없음)
🛡️
PRIVATE
BY DESIGN

보안은 기능이 아닌 설계의 원칙입니다. 나중에 추가하는 것이 아니라, 처음부터 구조 안에 녹아 있습니다.

경쟁사가 "안전합니다"라고 말할 때, 우리는 "안전하게 설계되어 있어서 원래 볼 수가 없습니다"라고 말합니다.

서비스의 완성은 기능이 아니라 사용자가 막히지 않는 것. 예상 가능한 모든 혼란을 미리 설계합니다.

HIGH

해결: 단계별 진행 상태 바 + 예상 소요 시간 실시간 표시, 완료 시 푸시 알림

HIGH
타임캡슐 봉인 후 취소 불가라는 사실을 몰랐음

해결: 봉인 전 최종 확인 모달 + 봉인 후 24시간 이내 취소 가능한 냉각 기간 제공

MED PRIORITY — 혼란 지점 · 경험 저하 위험
MED
분석 연구소 탭이 너무 많아 어디서 시작해야 할지 혼란

해결: 첫 진입 시 '추천 순서' 온보딩 투어 + 인기 탭 기본 활성화

MED

해결: 첫 업로드 완료 직후 '내 데이터 보호 방식 확인' 프라이버시 요약 카드 의무 노출

MED
백서 테마 선택지가 많아 결정 마비 발생

해결: 테마 5개 이내로 제한, AI 추천 테마 1개를 기본 선택 상태로 제시
