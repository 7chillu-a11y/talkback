# Talkback — 서비스 기획 발표 슬라이드

> 카카오톡 대화를 휘발되지 않는 자산으로 전환하는 AI 기반 아카이빙 서비스 기획안  
> HTML 슬라이드 형식 · GitHub Pages 배포

---

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | Talkback (톡백) |
| **형식** | 인터랙티브 HTML 슬라이드 |
| **주제** | 카카오톡 대화 아카이빙 서비스 기획 |
| **배포** | GitHub Pages |
| **제작자** | IP기획 전공 학생 |

**Talkback**은 카카오톡 내보내기 기능의 한계(낮은 가독성, 번거로운 절차, 보안 불안)를 해결하기 위해 기획된 로컬 기반 대화 아카이빙 서비스입니다. 드래그 앤 드롭만으로 대화를 안전하게 보존하고, 의미 있는 콘텐츠로 재구성하는 경험을 제안합니다.

---

## 핵심 기능

- **슬라이드 내비게이션** — 키보드(←/→) 및 버튼으로 슬라이드 전환
- **진행률 표시 바** — 상단 Progress Bar로 현재 위치 시각화
- **인쇄 최적화** — `@media print` 지원으로 PDF 변환 가능
- **반응형 레이아웃** — 다양한 화면 크기 대응
- **카카오 디자인 시스템** — 카카오 브랜드 컬러 기반 CSS Variables 적용

## 제작 방식 — Claude Agent 활용

이 프로젝트는 단순히 AI에게 결과물을 생성시킨 것이 아닙니다.  
**IP기획자로서 AI를 에이전트로 설계하고 디렉팅하는 방식**으로 제작했습니다.

### AI 리터러시 기반 협업 구조

```
기획자 (나)                    Claude Agent
────────────────────────────────────────────────────
문제 정의 & 방향 설정   ──►   리서치 & 데이터 분석
기획 의도 & 맥락 제공   ──►   슬라이드 구조 설계
피드백 & 수정 지시      ──►   HTML/CSS 구현
최종 판단 & 승인        ──►   인터랙티브 요소 구현
```

### Claude Agent가 기여한 영역

| 영역 | 구체적 기여 |
|------|------------|
| **서비스 기획** | 시장 조사, SWOT 분석, 사용자 Pain Point 도출 |
| **스토리텔링** | 슬라이드 흐름 설계 (문제 → 솔루션 → 검증) |
| **HTML/CSS 구조** | 시맨틱 마크업, CSS Variables, 전환 애니메이션 |
| **인터랙티브 요소** | 슬라이드 네비게이션, 진행률 바, 키보드 이벤트 |
| **디자인 시스템** | 카카오 브랜드 가이드라인 기반 컬러 팔레트 구성 |

### 기획자로서의 역할

AI가 생성한 초안에서 다음을 직접 판단하고 수정했습니다:
- 서비스 핵심 가치 명확화 및 방향 수정
- 슬라이드별 메시지 우선순위 재조정
- 사용자 심리(보안 불안, 진입 장벽)를 반영한 내러티브 보강
- 디자인 완성도 피드백 및 반복 개선 지시

> **AI 리터러시란 AI를 잘 쓰는 것이 아니라, AI에게 맡길 것과 내가 결정할 것을 구분하는 능력입니다.**

## 피드백 루프 (Feedback Loop)

> 이 프로젝트는 단 한 번의 프롬프트로 완성된 결과물이 아닙니다.  
> 기획자로서 방향을 설정하고, AI의 출력을 비판적으로 검토하고, 다시 구체적인 기준으로 수정을 지시하는 **전략적 피드백 사이클**을 반복하며 완성도를 높였습니다.

### 피드백 사이클 구조

```
① 기획 의도 설정
      ↓
② Claude Agent 초안 생성
      ↓
③ 기획자 검토 — "이 부분이 의도와 다르다"
      ↓
④ 수정 기준 명시 → 재생성 지시
      ↓
⑤ 결과 채택 / 재피드백
      ↓
    반복 →→→ 최종 결과물
```

### 실제 피드백 사례

#### 1. UX 중심의 슬라이드 인터랙션 고도화

초기 AI 생성 결과물은 슬라이드를 단순히 페이지 단위로 나열하는 구조였습니다.  
기획자로서 **"슬라이드 전환은 청중의 주의를 끊지 않아야 한다"** 는 UX 원칙을 기준으로 다음을 지시했습니다:

- 슬라이드 전환 시 `translateX` 기반 슬라이딩 애니메이션 적용 (단순 페이드 대비 방향성 인지 명확화)
- 상단 Progress Bar로 전체 흐름 중 현재 위치를 상시 인지할 수 있도록 요청
- 키보드(←/→) 네비게이션 추가 — 발표자가 마우스 없이 조작 가능하도록 설계
- 인쇄/PDF 변환 시 레이아웃 붕괴 방지를 위한 `@media print` 처리 별도 요청

> 단순한 "더 예쁘게 해줘"가 아니라, **발표 상황과 청중 경험을 구체적으로 전제한 뒤 기술 요건으로 변환**하여 지시한 것이 핵심입니다.

---

#### 2. IP기획자 관점의 기획 의도 반영

AI가 생성한 초안은 기능 나열 중심의 일반적인 서비스 소개에 머물렀습니다.  
IP기획 전공자로서 **"기획의 논리 구조가 슬라이드에 드러나야 한다"** 는 기준으로 다음을 수정 지시했습니다:

- 문제 정의 슬라이드에서 단순 불편 나열이 아닌, **사용자 심리(보안 불안, 진입 장벽)** 를 전면에 배치하도록 재구성 요청
- 솔루션 슬라이드에서 기능 소개 순서를 '사용자가 가장 먼저 느끼는 불안 해소 → 핵심 기능 제시' 순으로 재배열 지시
- SWOT 분석 결과를 슬라이드에 그대로 옮기지 않고, **기획 방향성과 연결되는 인사이트만 추려 요약**하도록 요청
- 카카오 브랜드 컬러 시스템을 단순 참조가 아닌, CSS Design Token으로 구조화하여 일관성을 유지하도록 기술 수준까지 지정

> 기획자의 피드백은 "이게 마음에 안 든다"가 아니라 **"왜 이 순서여야 하는가"에 대한 논리를 제시하는 것**이어야 합니다. 이 과정에서 AI는 실행 파트너로 기능했습니다.

---

### 피드백의 질이 결과물의 질을 결정한다

| 피드백 유형 | 예시 | 효과 |
|-------------|------|------|
| **모호한 요청** | "더 보기 좋게 해줘" | AI가 임의 해석 → 방향 불일치 |
| **기준 있는 요청** | "발표자 관점에서 키보드 조작이 가능해야 해" | 요건 명확 → 의도에 맞는 구현 |
| **맥락 있는 요청** | "보안 불안이 진입 장벽의 핵심이니까 이걸 첫 Pain Point로 올려줘" | 기획 논리 반영 → 설득력 있는 결과 |

> 이 프로젝트를 통해 확인한 것은, **AI 에이전트의 출력 품질은 프롬프트의 구체성과 기획자의 판단력에 비례한다**는 사실입니다.
