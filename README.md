# Talkback — 서비스 기획 발표 슬라이드

> 카카오톡 대화를 휘발되지 않는 자산으로 전환하는 AI 기반 아카이빙 서비스 기획안  
> HTML 슬라이드 형식 · GitHub Pages 배포

---

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | Talkback (톡백) |
| **형식** | 인터랙티브 HTML 슬라이드 |
| **주제** | 카카오톡 대화 아카이빙 서비스 기획 |
| **배포** | GitHub Pages |
| **제작자** | IP기획 전공 학생 |

**Talkback**은 카카오톡 내보내기 기능의 한계(낮은 가독성, 번거로운 절차, 보안 불안)를 해결하기 위해 기획된 로컬 기반 대화 아카이빙 서비스입니다. 드래그 앤 드롭만으로 대화를 안전하게 보존하고, 의미 있는 콘텐츠로 재구성하는 경험을 제안합니다.

---

## 핵심 기능

- **슬라이드 내비게이션** — 키보드(←/→) 및 버튼으로 슬라이드 전환
- **진행률 표시 바** — 상단 Progress Bar로 현재 위치 시각화
- **인쇄 최적화** — `@media print` 지원으로 PDF 변환 가능
- **반응형 레이아웃** — 다양한 화면 크기 대응
- **카카오 디자인 시스템** — 카카오 브랜드 컬러 기반 CSS Variables 적용

## 제작 방식 — Claude Agent 활용

이 프로젝트는 단순히 AI에게 결과물을 생성시킨 것이 아닙니다.  
**IP기획자로서 AI를 에이전트로 설계하고 디렉팅하는 방식**으로 제작했습니다.

### AI 리터러시 기반 협업 구조

```
기획자 (나)                    Claude Agent
────────────────────────────────────────────────────
문제 정의 & 방향 설정   ──►   리서치 & 데이터 분석
기획 의도 & 맥락 제공   ──►   슬라이드 구조 설계
피드백 & 수정 지시      ──►   HTML/CSS 구현
최종 판단 & 승인        ──►   인터랙티브 요소 구현
```

### Claude Agent가 기여한 영역

| 영역 | 구체적 기여 |
|------|------------|
| **서비스 기획** | 시장 조사, SWOT 분석, 사용자 Pain Point 도출 |
| **스토리텔링** | 슬라이드 흐름 설계 (문제 → 솔루션 → 검증) |
| **HTML/CSS 구조** | 시맨틱 마크업, CSS Variables, 전환 애니메이션 |
| **인터랙티브 요소** | 슬라이드 네비게이션, 진행률 바, 키보드 이벤트 |
| **디자인 시스템** | 카카오 브랜드 가이드라인 기반 컬러 팔레트 구성 |

### 기획자로서의 역할

AI가 생성한 초안에서 다음을 직접 판단하고 수정했습니다:
- 서비스 핵심 가치 명확화 및 방향 수정
- 슬라이드별 메시지 우선순위 재조정
- 사용자 심리(보안 불안, 진입 장벽)를 반영한 내러티브 보강
- 디자인 완성도 피드백 및 반복 개선 지시

> **AI 리터러시란 AI를 잘 쓰는 것이 아니라, AI에게 맡길 것과 내가 결정할 것을 구분하는 능력입니다.**
