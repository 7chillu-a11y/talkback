# Agent Log — Claude Agent 활용 기록

이 폴더는 Talkback 프로젝트 제작 과정에서 Claude Agent에게 전달한  
**프롬프트, 작업 지침, 피드백 로그**를 보관합니다.

---

## 이 폴더의 목적

AI를 단순히 사용한 것이 아니라 **에이전트로 설계하고 운용했다**는 사실을 기록합니다.  
기획자가 AI에게 무엇을 지시했고, 어떤 판단을 직접 내렸는지를 추적할 수 있습니다.

## 작업 원칙

| 원칙 | 설명 |
|------|------|
| **명확한 역할 분리** | AI는 생성, 기획자는 판단 — 역할을 명확히 구분합니다 |
| **컨텍스트 우선 제공** | 프롬프트 작성 전 프로젝트 배경과 제약 조건을 먼저 명시합니다 |
| **반복적 피드백** | 한 번의 생성으로 끝내지 않고 여러 차례 검토·수정합니다 |
| **가정 명시** | AI가 추론한 내용은 반드시 검토 후 채택 여부를 결정합니다 |
| **기록 유지** | 어떤 지시를 했는지 남겨 재현 가능한 작업 과정을 만듭니다 |

## 핵심 프롬프트 패턴

### 1. 역할 부여 (Role Prompting)
```
너는 IP기획 분야의 시니어 서비스 기획자야.
지금부터 [Talkback] 서비스의 기획안을 함께 만들 거야.
```

### 2. 컨텍스트 + 제약 조건 명시
```
- 타겟: 20~30대 카카오톡 헤비 유저
- 핵심 가치: 로컬 처리 기반 보안성
- 형식: HTML 슬라이드 (외부 라이브러리 없이)
- 디자인: 카카오 브랜드 컬러 시스템 준수
```

### 3. 단계별 분해 (Task Decomposition)
```
전체를 한 번에 만들지 말고, 다음 순서로 진행해:
1. 슬라이드 목차 구성 → 내 피드백 → 2. 각 슬라이드 콘텐츠 → 3. HTML 구현
```

### 4. 피드백 루프
```
[수정 요청] 3번 슬라이드 문제 정의 파트에서 '보안 불안' 요소를
더 강조해줘. 현재는 UX 이슈에 묻혀 있어.
```

## 기획자가 직접 결정한 사항

AI의 제안 중 다음 항목은 기획자가 직접 판단하여 수용/수정/기각했습니다:

- [ ] 서비스 핵심 가치 정의 (보안 vs 편의성 우선순위)
- [ ] 슬라이드 흐름 구조 (기각 후 재설계한 섹션 포함)
- [ ] 경쟁사 포지셔닝 방향
- [ ] 최종 디자인 컬러 시스템 채택 여부
- [ ] 인터랙티브 요소 범위 결정
